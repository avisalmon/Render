# babook.co.il — Research Summary & Next Steps

## What We Learned

### Market Landscape
- **8 competitors analyzed** across 4 segments: courses (DL.AI), hands-on labs (Kaggle), coding platforms (GitHub Copilot, Replit), and model hubs (Hugging Face)
- **Key gap:** No platform combines all 5 of Avi's scopes (multi-level training + community + token provisioning + service marketplace + meetups)
- **Opportunity:** babook can be the "all-in-one AI community platform" for Israel-first, then global

### Competitive Strengths of babook
1. **Integrated:** One place for learning, building, hiring, and monetizing
2. **Community-first:** Unlike DeepLearning.AI (content) or GitHub (tools), we lead with peer-to-peer
3. **Token control:** Unlike Kaggle/Coursera, we can resell GitHub Copilot + OpenAI credits as a feature
4. **Local advantage:** Hebrew-speaking + Israel-based network (cultural fit)
5. **Multi-sided marketplace:** Learners ↔ Educators ↔ Advisors ↔ Companies

### Market Segments Identified
| Segment | Pain Point | babook Solution |
|---------|-----------|------------------|
| Junior devs (100s in Israel) | Can't afford Copilot + training | Bundled skills + discounted Copilot |
| ML researchers (200s in Israel) | Isolated; no peer network | Community + mentorship + compute |
| AI agencies/startups | Need trained staff quickly | Corporate training + hiring board |
| Self-taught learners | Too many resources, no path | Structured tracks by level |
| Freelance advisors | Hard to find clients | Marketplace + credibility badges |

## Key Research Questions (for Avi)

Before we lock in Chapter 2 scope, we need answers to:

### 1. **Which primary user are we serving first?**
   - A) Self-taught learners (user level → practical level)
   - B) Practitioners building AI products (practical level focus)
   - C) ML researchers (deep level focus)
   - D) AI teams at Israeli companies (training + hiring)

**Why:** Each requires different product focus:
- A = heavy video + gamification + forums
- B = IDE + labs + skill marketplace
- C = compute + research forums + paper discussions
- D = corporate dashboard + group licensing + hiring

### 2. **What's our revenue model primary focus?**
   - A) Subscriptions (courses)
   - B) Marketplace (advisors, skills, tokens)
   - C) Token resale (GitHub Copilot, OpenAI credits)
   - D) Corporate training deals

**Why:** Determines feature priority and initial user acquisition strategy.

### 3. **Which scope should be MVP?**
   - A) Training (videos + labs)
   - B) Community (forums + skill store)
   - C) Tokens (Copilot resale + credits)
   - D) Services (advisor marketplace)

**Why:** Each has different go-to-market complexity:
- A = 2-3 month course dev per skill
- B = 2 weeks infrastructure + moderation setup
- C = 1 week if we partner with Copilot/OpenAI
- D = requires 5-10 vetted advisors to launch

### 4. **Geographic/language focus?**
   - A) Israel-only (Hebrew-first, English secondary)
   - B) Hebrew-speaking diaspora (Israel + diaspora)
   - C) English-speaking global (English-first, Hebrew optional)

**Why:** Affects community tone, marketing, partnerships, and growth trajectory.

### 5. **Live vs asynchronous?**
   - A) Video courses + recorded sessions (async-first)
   - B) Live coding + weekly cohorts (sync-first)
   - C) Mix (recorded courses + live labs)

**Why:** Affects team size, time zone complexity, and learner commitment level.

## Recommendation: Prioritized Roadmap

Based on market research, **I recommend this MVP stack (Avi, please validate):**

### **Primary User:** Self-taught + junior devs (lowest barrier to entry)
### **Primary Revenue:** Token resale + Skill marketplace
### **Primary Scope:** Practical Level (co-coding, agents, MCP)

**Rationale:**
1. **Practitioners have money** (paying for Copilot already) → token resale immediate revenue
2. **Skill marketplace is easier to launch** than full courses (creators drive content)
3. **Community forums are free retention** (keep users engaged between paid tiers)
4. **Hands-on labs beat video** for practitioners (learn by doing)
5. **Israeli dev community is mature** (ready for practical content)

### **Proposed Chapter 2 Scope (MVP)**

| Feature | Phase | Owner | Timeline |
|---------|-------|-------|----------|
| **Training** |
| 3-5 practitioner skill tracks | Phase 1 | Copilot | 4 weeks |

| GitHub Copilot integration | Phase 1 | Copilot | 1 week |
| **Community** |
| Forums (Q&A + discussions) | Phase 1 | Copilot | 2 weeks |
| Project showcase | Phase 1 | Copilot | 2 weeks |

| **Tokens** |
| GitHub Copilot resale | Phase 2 | Avi | 1 week (partnership) |

| Weekly newsletter | Phase 2 | Copilot | 1 week |

**Total MVP:** 6-8 sprints (12-16 weeks)

### **Key Success Metrics (MVP)**
- 500 active users by end of Phase 1
- 100+ created skills in marketplace
- 1,000+ forum posts
- ₪50K+ token resale revenue (Phase 2)

## What NOT to Do (Avoid)

1. ❌ **Don't copy Kaggle** — competitions are low-engagement for most learners
2. ❌ **Don't do video courses first** — 12 weeks per course, slow to market
3. ❌ **Don't launch with advisors** — need critical mass of users first
4. ❌ **Don't target researchers first** — small market, high compute costs
5. ❌ **Don't go global immediately** — Israel is test market, validate product-market fit first
